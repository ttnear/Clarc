[2026-04-20] 문제: InputBarView paste 로직이 handlePasteKey + onChange 2곳에 분산. onChange 감지가 delta>1 + (isPasteInProgress || pbCount != lastPasteChangeCount || clipboardHasImageOrFile) 같은 휴리스틱에 의존해, 이미지 paste 후 한글 IME + space 커밋 시 delta>1 되면 같은 이미지가 중복 첨부되고 입력 텍스트가 롤백됨. → 해결: handlePasteKey를 단일 진입점으로 만들어 Cmd+V를 항상 .handled로 소비. clipboard 내용 우선순위(fileURL → image data → URL → long text → short text)로 분기. onChange 감지 로직, isPasteInProgress, lastPasteChangeCount, skipPasteDetection state 전부 제거. 교훈: paste 감지를 "텍스트 변경 delta" 같은 후행 휴리스틱으로 역추적하지 말고, 실제 paste 이벤트(Cmd+V / drag-drop)에서 직접 처리할 것.

[2026-04-20] 문제: assistant 텍스트 버블이 한 줄로 잘려 "..." 표시됨. MessageBubble.swift의 `assistantTextBubble`는 HStack(alignment: .bottom)으로 텍스트와 streaming cursor를 감싸는데, streaming Text에는 `.frame(maxWidth: .infinity, alignment: .leading)`이 있어 width 확장되지만 non-streaming의 MarkdownContentView에는 누락되어 있어 HStack이 intrinsic width로 측정되어 truncate됨. → 해결: MarkdownContentView에도 동일한 `.frame(maxWidth: .infinity, alignment: .leading)` 추가. 교훈: HStack의 자식 중 하나라도 width-expanding modifier가 빠지면 두 갈래(streaming/non-streaming)에서 layout 동작이 달라짐. 분기되는 뷰의 layout modifier는 동일하게 맞출 것.
